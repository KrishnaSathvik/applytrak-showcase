## üèóÔ∏è Architecture & Design Decisions

### **Component Architecture**
```
src/
‚îú‚îÄ‚îÄ components/
‚îÇ   ‚îú‚îÄ‚îÄ charts/          # Recharts analytics components
‚îÇ   ‚îú‚îÄ‚îÄ forms/           # React Hook Form components with validation
‚îÇ   ‚îú‚îÄ‚îÄ layout/          # Header, sidebar, responsive layout components
‚îÇ   ‚îú‚îÄ‚îÄ modals/          # Goal setting, application edit dialogs
‚îÇ   ‚îú‚îÄ‚îÄ tables/          # Paginated, searchable data tables
‚îÇ   ‚îî‚îÄ‚îÄ ui/              # Reusable design system components
‚îú‚îÄ‚îÄ hooks/               # Custom React hooks for business logic
‚îú‚îÄ‚îÄ services/            # Database services (Dexie + Supabase)
‚îú‚îÄ‚îÄ store/               # Zustand state management stores
‚îú‚îÄ‚îÄ types/               # TypeScript type definitions
‚îî‚îÄ‚îÄ utils/               # Helper functions, formatters, constants
```

### **Key Design Principles**
- **Component Reusability**: Every UI element is a reusable component
- **Type Safety**: 100% TypeScript coverage with strict mode enabled
- **Performance First**: Lazy loading, memoization, and virtualization
- **Accessibility**: WCAG 2.1 compliant with keyboard navigation
- **Mobile-First**: Responsive design starts with mobile constraints

### **State Management Strategy**
- **Zustand Stores**: Separate stores for applications, goals, UI state, and settings
- **Local Persistence**: Automatic persistence with IndexedDB
- **Optimistic Updates**: UI updates immediately, syncs in background
- **Error Boundaries**: Graceful error handling with recovery options
